# Proto Regeneration

`server/proto/service.proto` is the only editable protobuf contract source in this repository. The checked-in client `service.proto` copies and generated outputs listed below are tracked artifacts that must be regenerated from the server-owned source rather than hand-edited in place.

## Canonical Commands

- Regenerate all tracked proto artifacts: `cd server && make gen-proto-all`
- Verify that tracked proto artifacts are up to date: `cd server && make check-proto-drift`

`make gen-proto-all` is the canonical full-regeneration entrypoint for this repository. It regenerates the server protobuf outputs and the tracked client artifacts that are checked into the Go, Python, and TypeScript SDK folders.

## Tracked Artifacts

These paths are checked-in synced copies or generated outputs that must stay synchronized with `server/proto/service.proto`.

### Editable Source (Authoritative)

- `server/proto/service.proto`

### Synced Input Copies (Read-Only)

These files are copied from the server by `generate_client_protos.sh` and carry a `// SYNCED COPY` header. Do not edit them directly.

- `clients/go-client/proto/service.proto`
- `clients/go-client/google/api/annotations.proto`
- `clients/go-client/google/api/http.proto`
- `clients/typescript-client/proto/service.proto`
- `clients/typescript-client/google/api/annotations.proto`
- `clients/typescript-client/google/api/http.proto`

### Generated Outputs (Read-Only)

These files are regenerated by `protoc` or language-specific tooling and must not be edited by hand.

- `server/proto/service.pb.go`
- `server/proto/service_grpc.pb.go`
- `server/proto/service.pb.gw.go`
- `clients/go-client/proto/service.pb.go`
- `clients/go-client/proto/service_grpc.pb.go`
- `clients/python-client/toolplane/proto/service_pb2.py`
- `clients/python-client/toolplane/proto/service_pb2_grpc.py`
- `clients/typescript-client/src/proto/`

`server/client_protos/` is not an authoritative checked-in output surface and should not be used as the source of truth for client wrapper updates.

## Toolchain Reference

The drift workflow installs the following toolchain family so CI and local regeneration have a documented baseline:

- `protoc` 21.12 (`libprotoc 3.21.12`)
- `protoc-gen-go` v1.36.6
- `protoc-gen-go-grpc` v1.5.1
- `protoc-gen-grpc-gateway` v2.26.3
- `grpcio` v1.78.0
- `grpcio-tools` v1.78.0
- `protobuf` v6.31.1
- Node dependencies from `clients/typescript-client/package-lock.json`, including `grpc-tools` and `grpc_tools_node_protoc_ts`

## Regeneration Flow

1. Regenerate the server Go and gRPC-gateway outputs from `server/proto/service.proto`.
2. Sync the checked-in client `service.proto` copies and the minimal `google/api` import copies used by the Go and TypeScript clients.
3. Regenerate the tracked Go client stubs in `clients/go-client/proto/`.
4. Regenerate the tracked Python client stubs in `clients/python-client/toolplane/proto/`. The follow-up import-fix step normalizes the generated gRPC stub so it uses the package-relative `toolplane.proto` layout instead of depending on a top-level `proto` package.
5. Regenerate the tracked TypeScript JavaScript and declaration outputs in `clients/typescript-client/src/proto/` via `npm run generate:proto`.

## JavaScript And TypeScript Outputs

The repository does not maintain a separate `server/client_protos/javascript/` tree. The checked-in JavaScript-facing protobuf outputs for the TypeScript package live under `clients/typescript-client/src/proto/` and are generated from the local `clients/typescript-client/package.json` script.
